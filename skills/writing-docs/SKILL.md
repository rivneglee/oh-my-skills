# Writing Docs

Write documentation that helps a future reader take correct action.

## Process

1. Identify the reader and the job they need to complete.
2. Inspect the related code, config, or existing docs before writing.
3. Prefer concrete steps, commands, paths, and examples over general advice.
4. Remove stale or duplicate content when updating existing docs.
5. Run available validation or examples when docs include commands.

## Output Standards

- Put the most common path first.
- Use headings that describe the task or concept.
- Keep setup, usage, troubleshooting, and reference material separate.
- Avoid claims that were not verified from the repository or a cited source.
